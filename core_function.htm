core.Function    (native)  
    Pops extends the native JS function type. Adding many useful features to the base and instances.

File  
    pops/pops.core.js

Syntax  
var foo = new Class(specs , onLoaded  );
var foo = Class(specs  ,onLoaded  );
  • specs : An object containing the specifications for the new class.
  • onLoaded : optional - The function to be called after the new class is created.

Members  
  • Bind: Returns a copy of the function, where the this object is the object specified by the obj argument.  
    var bar = foo.Bind(obj);
    • bar : This will reference oo.
    • foo : Any function.
    • obj : Any object.
  • aApply: Calls the function on the next tick of the event loop. Accepts an array(arguments) to pass to the function. Returns parent-function.  
    var bar = foo.aApply(arg, _this);
    foo = foo.aApply(arg, _this);
    foo.aApply(arg, _this);
    • bar : This will reference foo.
    • foo : Any function.
    • args : An array or arguments-object. The contents will be passed to the function when it is called.
    • _this : An object to be used as the this object by the function when it is called. If omitted, the function will use it's normal this object.
  • aCall: Calls the function on the next tick of the event loop. Returns parent-function.  
    var bar = foo.aCall(arg1, arg2);
    foo = foo.aCall(arg1, arg2);
    foo.aCall(arg1);
    • bar : This will reference foo.
    • foo : Any function.
    • arg1, arg2 : These will be passed as arguments to foo, when called. You can put an unlimited number of arguments here.
  • Reg: Registers the function to the value-registry. Returns parent-function.  
    var foo = function() { /* code */ }.Reg(unique_name);
    bar = bar.Reg(unique_name);
    bar.Reg(unique_name);
    • foo : This will reference the function.
    • bar : Any function.
    • unique_name : A string used to identify this function. when you go to retrieve this function from the value-registry, you can use this value as the id argument.
  • Str: Returns a string representation of the function. Same as toString.  
    var foo = function() { /* code */ }.Str();
    • foo : This will be a string representing the parent-function.
  • Private: Makes the function private when included in a Class. Returns parent-function.  
    var bar = foo.Private();
    foo = foo.Private();
    foo.Private();
    • bar : Will contain a reference to foo.
    • foo : Any function.
  • Protected: Makes the function protected when included in a Class. Returns parent-function.  
    var bar = foo.Protected();
    foo = foo.Protected();
    foo.Protected();
    • bar : Will contain a reference to foo.
    • foo : Any function.
  • Shared: Makes the function shared when included in a Class. Returns parent-function.  
    var bar = foo.Shared();
    foo = foo.Shared();
    foo.Shared();
    • bar : Will contain a reference to foo.
    • foo : Any function.
  • PrivShare: Makes the function private and shared when included in a Class. Returns parent-function.  
    var bar = foo.PrivShare();
    foo = foo.PrivShare();
    foo.PrivShare();
    • bar : Will contain a reference to foo.
    • foo : Any function.
